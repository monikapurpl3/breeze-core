Breeze Core packages

Signed packages for Breeze Core — self-hosted, LAN-first control for Midea air conditioners. Current version @VERSION@.

How this works: add the repository once with the snippet for your system below, then install and update through your normal package manager — apt upgrade, dnf upgrade, zypper up, pacman -Syu, apk upgrade. Everything is built and signed offline; this server only serves static files and holds no keys, so verify against the published keys. amd64/x86_64 and arm64/aarch64 (BSD: amd64 only).

Jump to your system:
Debian / Ubuntu Fedora / RHEL openSUSE / SLE Arch Alpine OpenWrt FreeBSD NetBSD After install

Debian family Debian · Ubuntu · Mint · Pop!_OS · Raspberry Pi OS · Devuan — apt

curl -fsSL https://bolero.salataputarica.hr.eu.org/breeze-core.asc \
  | sudo gpg --dearmor -o /usr/share/keyrings/breeze-core.gpg
echo "deb [signed-by=/usr/share/keyrings/breeze-core.gpg] https://bolero.salataputarica.hr.eu.org/deb stable main" \
  | sudo tee /etc/apt/sources.list.d/breeze-core.list
sudo apt update && sudo apt install breeze-core

Fedora / RHEL family Fedora · RHEL · AlmaLinux · Rocky · CentOS Stream — dnf

Import the signing key first (works the same on every version), then add the repo:

sudo rpm --import https://bolero.salataputarica.hr.eu.org/breeze-core.asc
sudo tee /etc/yum.repos.d/breeze-core.repo <<'EOF'
[breeze-core]
name=Breeze Core
baseurl=https://bolero.salataputarica.hr.eu.org/rpm/$basearch/
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://bolero.salataputarica.hr.eu.org/breeze-core.asc
EOF
sudo dnf install breeze-core
dnf 4 vs dnf 5. The .repo file above works on both. On dnf 4 (RHEL 8/9, Fedora ≤ 40) the key is imported interactively on the first install — the rpm --import step just does it up front so there's no prompt. On dnf 5 (Fedora 41+, RHEL 10) that pre-import is required: dnf 5 no longer pulls the key from gpgkey= on its own. If you prefer dnf config-manager, note the verb changed too — dnf 5 wants dnf config-manager addrepo --from-repofile=<url> (dnf 4 used --add-repo).

openSUSE / SLE Leap · Tumbleweed · SLES/SLED — zypper

Same rpms, but zypper adds and refreshes the repo its own way:

sudo rpm --import https://bolero.salataputarica.hr.eu.org/breeze-core.asc
sudo zypper addrepo --refresh --gpgcheck \
  https://bolero.salataputarica.hr.eu.org/rpm/x86_64/ breeze-core
sudo zypper --gpg-auto-import-keys refresh
sudo zypper install breeze-core

On aarch64 use rpm/aarch64/ in the URL. --gpg-auto-import-keys accepts the repo signature non-interactively.

Arch family Arch · Manjaro · Artix — pacman

curl -fsSL https://bolero.salataputarica.hr.eu.org/breeze-core.asc | sudo pacman-key --add -
sudo pacman-key --lsign-key "$(gpg --show-keys --with-colons \
  <(curl -fsSL https://bolero.salataputarica.hr.eu.org/breeze-core.asc) | awk -F: '/^fpr/{print $10; exit}')"
sudo tee -a /etc/pacman.conf <<'EOF'

[breeze-core]
SigLevel = Required
Server = https://bolero.salataputarica.hr.eu.org/arch/$arch
EOF
sudo pacman -Sy breeze-core

Alpine apk

sudo curl -fsSL -o /etc/apk/keys/breeze-core@bolero.rsa.pub \
  https://bolero.salataputarica.hr.eu.org/alpine/breeze-core@bolero.rsa.pub
echo "https://bolero.salataputarica.hr.eu.org/alpine" | sudo tee -a /etc/apk/repositories

OpenWrt x86_64 / aarch64 routers with ~65 MB free (extroot counts) — opkg

wget -q -O /tmp/breeze.pub https://bolero.salataputarica.hr.eu.org/openwrt/breeze-core-usign.pub
cp /tmp/breeze.pub "/etc/opkg/keys/$(usign -F -p /tmp/breeze.pub)"
echo "src/gz breeze_core https://bolero.salataputarica.hr.eu.org/openwrt/$(. /etc/openwrt_release; echo $DISTRIB_ARCH)" \
  >> /etc/opkg/customfeeds.conf
opkg update && opkg install breeze-core
# then: breeze-core pair && /etc/init.d/breeze-core start   (procd service)

FreeBSD amd64 — a real, RSA-signed pkg repo

sudo mkdir -p /usr/local/etc/pkg/keys /usr/local/etc/pkg/repos
sudo fetch -o /usr/local/etc/pkg/keys/breeze-core.pub \
  https://bolero.salataputarica.hr.eu.org/freebsd/breeze-freebsd-repo.rsa.pub
sudo tee /usr/local/etc/pkg/repos/breeze-core.conf <<'EOF'
breeze-core: {
  url: "https://bolero.salataputarica.hr.eu.org/freebsd",
  signature_type: "pubkey",
  pubkey: "/usr/local/etc/pkg/keys/breeze-core.pub",
  enabled: yes
}
EOF
sudo pkg update && sudo pkg install breeze-core
# then: breeze-core pair && sysrc breeze_core_enable=YES && service breeze_core start

NetBSD amd64/x86_64 — a pkgin binary repo (needs python312 from pkgsrc)

echo "https://bolero.salataputarica.hr.eu.org/netbsd/All" \
  | su root -c 'tee -a /usr/pkg/etc/pkgin/repositories.conf'
su root -c 'pkgin update && pkgin -y install breeze-core'
# then: breeze-core pair && echo breeze_core=YES >> /etc/rc.conf && service breeze_core start

BSD builds from a private virtualenv (the Linux self-contained bundles can't run on a BSD kernel). Verified on real FreeBSD 15 and NetBSD 10. OpenBSD: build from source — see PACKAGES.md.

After installing any system — three steps to a running server

# 1. sudo breeze-core pair                        # find your ACs, write the config
# 2. sudoedit /etc/breeze-core/breeze-core.env    # set BREEZE_HOST=<this machine's LAN IP>
# 3. sudo systemctl enable --now breeze-core      # OpenRC: rc-update add breeze-core default; BSD: see above

Then open http://<BREEZE_HOST>:8420, enter the API key from /etc/breeze-core/config.json, and pair. The service runs as an unprivileged breeze user. Uninstalling keeps your config + device tokens.

Keys: GPG (apt/dnf/zypper/pacman) · RSA (apk) · RSA (FreeBSD pkg) · usign (OpenWrt).
